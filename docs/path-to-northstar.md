# 通往北极星(8–15 岁少年智力)的路径匹配分析

> 本文档基于社区常见的"五条 AI 路线"分析，结合 karbon 代码库**真实状态**做校准，
> 给出最契合本项目基因、且不违反 `PLAN.md` 任一 Non-Goal 的推进路线。
>
> 北极星定义见 `PLAN.md §1.5`：**人类 8–15 岁少年智力，必须从零、自主、发育式达成**。
> 铁律见 `AGENTS.md §2 规则#0`：所有方向以"最有利于终点的路"为准，不取省力/降标捷径。

## 0. 五条路线速览（社区视角）

| 路径 | 本质 | 达 8–15 岁潜力 | 与项目契合 | 代价 |
|---|---|---|---|---|

## 1. 对社区分析的**代码事实校准**（重要）

社区原分析有几处与 karbon 实际代码不符，需修正后再决策：

### 1.1 "imagine_step 只是没真用" —— **不准确**
- `src/train.py:1989` 的 `wm.imagine_step` 仅用于 **RSSM 不确定性好奇心奖励**
  （预测下一帧误差当 intrinsic reward），**不是** Dreamer 式想象训练。
- 真正的 Dreamer 式想象训练在 `src/training/imagination_trainer.py`，由
  `train.py:2630` 的 `ImaginationTrainer.train_step(actor_critic=model, ...)`
  **调用，且梯度回传主模型**——即**已正确接线进训练闭环**。
- 它仅被 `if imagination_cfg and enabled` 门控。**Stage 5 的 yaml 没开
  `imagination.enabled`**，所以当前是"一键待激活"状态，**并非缺失/未实现**。
- **含义**：路径 2 的补完在 karbon 里几乎是零工程量（开开关即可），
  远比"重构世界模型"乐观。

### 1.2 "参数扩到 500M–1B" 与 Axiom 1（有界）冲突
- `AGENTS.md` 铁律第 3 条：永不创建无界 GPU 结构；PLAN Non-Goals 限定规模。
- 扩容可行，但必须用**有界容量 + 分层存储（SSD/冷层）**实现，不能简单堆参数。
- **已决策解法（A+B 组合，见 §1.6）**：不靠堆单模型参数，而是用世界模型想象
  提升样本效率 + 有界分层外部记忆放大有效智能体量，在 Axiom 1 内逼近 10 岁级。

### 1.6 规模 vs Axiom 的官方解法：A+B 组合（已选定）

目标：在**不违反 Axiom 1（有界 / 容量声明）**的前提下，把"有效智能体量"
放大一个数量级，逼近路径 2 所需的 10 岁级物理直觉。

- **方案 A · 有界分层外部记忆（项目已有范式）**
  - 不堆单模型参数，把可寻址知识分散到分层存储：GPU 热层（有界，如 256 技能）
    + CPU 温层 + SSD 冷层（`bounded_replay.py` / `skill_library.py` 已是此范式）。
  - 模型本体保持有界（150M–500M），但"总智能"通过**检索式注入**放大——
    即 Stage 4 的 LoRA 技能库思路推广为通用外部记忆。
  - 契合度：✓✓✓ 完全不违反 Axiom 1，是项目基因延伸。
  - 代价：推理时需检索 + 注入，latency 增加；检索与在线推理的耦合仍需工程打磨。

- **方案 B · 世界模型想象替代参数（Dreamer 路线精髓）**
  - 小模型（150M 级）+ 高质量 RSSM 世界模型，用"想得多"替代"参数多"：
    想象里练 100 步 ≈ 环境交互 1 步，样本效率提升直接抵消参数劣势。
  - DreamerV3 论文实证：150M 级 RSSM 能在数十任务超人类基线，靠想象 rollout
    而非参数规模。
  - 契合度：✓✓✓ 即 `stage6_consolidation.yaml` 已开的 `imagination.enabled`，
    零成本起步。
  - 代价：世界模型质量决定上限；PhysicSandbox 视觉 WM 若不够准，想象会塌。

- **结论**：A+B 组合为官方解法。B 已在 Stage 6 激活，先把这条跑透；
  A 的"检索式外部记忆注入"作为后续深化方向（技能库 → 通用认知记忆）。
  备选方案 C（有界 MoE）与方案 D（放宽规模约束，需用户批准改铁律）暂不采用。

> 注：C（MoE 但有界，专家数 × 每专家参数声明上限）与 D（重新定义 Axiom 边界）
> 仍保留为后备，仅在 A+B 被实证不足以逼近目标时再评估。

### 1.3 "换 Brax/MuJoCo" 会破坏发育主链
- 当前 PhysicsSandbox 是 64×64 图像连续控制，obs 形状固定、难度只调物理参数——
  这正是"从零发育"可控的关键。
- 换成 MuJoCo 状态向量输入 = 换感知范式，会与 Stage 4/5 已累积的视觉编码器 /
  slot 表征**断链**。
- 更契合的升级是项目已搭好的 `src/envs/three_d_world.py` /
  `extended_3d_world.py`（仍是图像 obs），而非外部 MuJoCo。

### 1.4 "NeuralSymbolicLayer 实现方式错（余弦匹配 ≠ 合一）" —— **判断准确，但有解法**

原分析把两个问题混为一谈，需拆开：

- **问题 Y · karbon 里真符号推理还没起步**（只有余弦匹配的
  `neuro_symbolic_bridge.py`，没有真 Prolog / miniKanren 后端）。
  这是**工程空白，不是科学难题**——真符号引擎（Prolog / miniKanren / Z3 /
  定理证明器）是数十年成熟成品，缺的只是"在 karbon 里接一个真后端 + 喂数据"。
- **问题 X · 神经↔符号接口（让梯度流过离散符号）仍是 30 年未解**。
  这是开放科学难题，**无根治解**（离散符号无法严格可微且保真）。

**问题 Y 的解法（可立即动手，不依赖 X）：**
- **Y1 · 外部符号引擎 + 神经只做感知/翻译（推荐起步）**：神经网络负责从
  slot/obs 抽谓词（`RuleInduction` 已在做）、把目标翻成逻辑查询；符号引擎
  （外部，如 `kanren` / `clplog`）负责真合一、推理、证伪。**梯度不穿过符号
  引擎**——神经部分用 REINFORCE / 行为克隆从符号结果学，绕开 X。
  - 后端选型建议：**`kanren`（miniKanren 的 Python 实现）做关系推理 /
    逻辑 unification；`clplog` 或 `swipl` 做规则证明；受限数值/约束域用
    `Z3`**。优先 `kanren`——轻量、纯 Python、易嵌进训练进程。
  - 首个验证域建议：**物理谜题（PhysicsSandbox 的因果/堆叠约束）或简单代数
    方程求解**——域窄、符号后端小且稳，接口问题被域的狭窄性压住。
- **Y2 · 可微符号（differentiable relaxation）**：用软合一 / 神经定理证明器
  （GPT-f / Tree-of-Thoughts 风格）近似，梯度可微。近似丢严格性，但工程能跑、
  能出 15 岁级受限域结果（AlphaGeometry 即此思路）。
- **Y3 · 程序归纳（Program Synthesis，项目已有 `program_synthesis.py`）**：
  把成功轨迹抽象成可复用"程序"，本身就是符号化且不要求经典逻辑后端
  （Lake 2015 单样本学习即此路线）。

**问题 X 的解法（无根治，但可工程绕开，业界实证有效）：**
- X1：不强行让梯度过符号——用 Y1 的"神经翻译 + 外部引擎 + 强化学回"，X 根本不发生。
- X2：放宽严格性换可微（Y2）。
- X3：只在受限域做（Y3 + AlphaGeometry 式），符号后端够小够稳，接口问题被压制。
- **关键认知**：业界 15 岁级成果（AlphaGeometry、AlphaProof）全都是**绕开 X**
  做的，不是解决了 X。故"真符号推理在受限域达到 15 岁级"是**可及的**，不用
  等 X 被解决。

**落地建议**：Stage 6 后支线用 Y1 起步——接 `kanren` 真后端到现有
`RuleInduction` / `CausalDiscovery` 输出，首个域选物理谜题或简单代数，验证
"神经抽谓词 → 外部引擎推理 → 结果回训神经"闭环。不触碰 X 的硬科学。

## 2. 最契合北极星的路线：组合拳（非二选一）

学界共识（LeCun JEPA + World Model 蓝图、DeepMind 内部路线）：

```
路径 2 (Dreamer 世界模型)      做 "直觉 / 感知 / 运动"
   +
路径 3 (真 Neuro-Symbolic)     做 "逻辑 / 数学 / 因果"
   +
路径 4 (Core Knowledge 先验)   做 "初始化 / 归纳偏置"
   +
少量 LLM (仅冻结编码器)        做 "文化知识锚点"
```

这套组合**不违禁**（无大模型预训练底座、发育式、预算可控），且是 karbon 基因
最能延伸的方向。

## 4. 关键工程优先级（据校准结论修正）

1. **第一优先**：Stage 6 开 `imagination.enabled` —— 零工程量，直接补完路径 2。
   （此前误以为需重构世界模型，实际只需开开关。）
2. **第二优先**：B 方案的 5 个轻量认知模块接 loss 闭环（HomeostaticDrives /
   Metacognition / LongRangePlanner / CausalDiscovery / CreativityOrchestrator）。
 3. **第三优先（Stage 6 后 Step 3）**：Core Knowledge 注入 **P1+P2 组合**——
    P1 程序化演示灌有界回放、P2 可微辅助 loss 惩罚先验违反（客体永存 / 直觉物理 /
    数感），均复用现有机制，无需等学界突破；P3 强化 slot/数感头既有偏置。
 4. **第四优先（Step 4）**：神经符号接口从"余弦匹配"升级为轻量真符号后端
    （Y1：外部 kanren 引擎 + 神经抽谓词 + 强化学回），受限域验证。
 5. **A 解法深化（与 B 并行）**：把 Stage 4 的 LoRA 技能库推广为**通用有界
    分层外部记忆**（GPU 热 / CPU 温 / SSD 冷），用检索式注入放大有效智能体量，
    避免单模型参数触顶 Axiom 1。B（想象训练）已在 Stage 6 激活，A 作为后续
    配套深化。
 6. **长线**：冻结多模态基础模型初始化（DINOv2 / CLAP 类，非 LLM）。

## 5. 最诚实的一句话

8–15 岁在任何路线下都是硬目标。要的不是"避开 LLM"，而是"找到不需要 LLM
也能填平那几道门（物理直觉 / 长程规划 / 符号推理 / 语言）的路径"。目前学界最有
希望的答案是 **世界模型 + 神经符号 + 核心先验** 的组合——而这恰好是当前 karbon
项目基因最能延伸的方向。注意"大"应体现在**世界模型深度 + 符号接口 + 先验注入**，
而非简单堆参数到 1B（那违反 Axiom 1）。
