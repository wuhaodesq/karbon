# Devagi Developmental Plan · 发育式智能体训练计划

> 从新生感知运动到学前儿童级智能（诚实预期：~3–5 岁等效，天鹅级）
> 2026-07-09 · 替代原 Stage 0–8 人工编排方案

---

## 0. 诚实声明

**当前 AI 无法训练出 15 岁人类智力水平。** 全世界最好的系统（GPT-4、Claude）用了万亿参数和互联网全量文本，也只在部分认知任务上达到成人水平。一个起始 1.5M 参数的 TTT-Hybrid 智能体不可能在当前技术条件下达到青少年级通用智力。

**但我们可以做一个严肃的尝试：** 用发育式范式（developmental AI），从新生感知运动出发，逐相开放能力，最终达到"能在一个复杂 3D 环境中完成多步推理任务、模仿简单语言指令、从经验中学习新技能"的水平。

这个水平的真实参照：不是 15 岁人类，是**一只聪明的乌鸦** —— 能使用工具、能解决多步因果问题、能从观察中学习。

---

## 1. 核心范式转换

| 旧方案（Stage 0–8）| 新方案（Phases 0–8）|
|---|---|
| 手动添加模块 | 同一架构在不同数据上生长 |
| 固定环境 MiniGrid | 环境随智能体复杂度一起增长 |
| 单一目标（return）| 发育式内在动机替换 |
| 人工 Stage 开关 | 生物启发的关键期（critical period）|
| CNN encoder | Slot Attention 从零学物体表征 |

---

## 2. 架构（不变的核心 + 三个新部件）

保留不改：
- **TTT-Hybrid 骨干**（TTT-Linear + SWA + FFN）— 支持 Model Growth 逐步扩大
- **RSSM 世界模型** — 预测下一个状态
- **Bounded Skill Library** — GPU/CPU/SSD 三层 LoRA 技能库
- **Bounded Replay Buffer** — 三层回放
- **Sleep Consolidation** — 周期性固化
- **EWC + GR VAE** — 防遗忘
- **六条有界公理** — 全程不变

新增：

| 部件 | 参数 | 作用 | 从哪个 Phase 开始 |
|---|---|---|---|
| **Slot Attention** | ~500K | 物体中心表征（scene → N 个槽位 + 属性）| Phase 1 |
| **Model Growth** | 0→150M | 发育式扩大骨干（加层/加头）| Phase 2+ |
| **Hierarchical Policy** | ~200K | 子目标 → 动作两层规划 | Phase 3+ |
| **Developmental Clock** | 0 | sin(发育年龄) 控制各层解冻时序 | Phase 0+ |

**关键约束：所有部件都在六条有界公理下运行。参数增长有上限（Phase 8 ≤ 150M）。**

## 3. 发育时钟（Developmental Clock）

不是手动 Stage 开关，而是一个连续发育信号：

```python
developmental_age = steps / 完整一步发育周期
# 不同子系统的"打开窗口"由正弦信号控制:
visual_openness = sigmoid((age - visual_window_start) / window_width)
motor_openness = sigmoid((age - motor_window_start) / window_width)
language_openness = sigmoid((age - language_window_start) / window_width)
# 对应层的 learning_rate *= openness
```

**关键期不"添加模块"，而是"允许已有模块开始学习"。** 视觉编码器从一开始就在模型中，但前 50k 步的学习率 = 0（冻结）。之后逐渐解冻。

## 4. 九阶段发育路线图

### Phase 0 · 新生（Protozoan）
**发育年龄**: 0–1 月 ｜ **参数**: 1.5M ｜ **训练**: ~1 周 cloud 24G

**能力目标**:
- 学会本体感觉：知道自己的动作改变了什么
- 学会物体永存：被遮挡的东西还在
- 学会基础物理：碰撞、重力

**环境**: 2D 物理沙盒（pymunk/Box2D）
- 1–3 个彩色物体（球、方块）
- 智能体是一个可移动的小圆
- 能碰、推、抓物体

**训练信号**:
- RND curiosity（随机 → 熟悉了就无聊）
- 世界模型预测损失（预测错了 → 这个状态值得探索）
- 物体交互奖励（碰了东西 + 物体移动了 → 正信号）

**发育控制**:
- Slot Attention 未解冻（视觉 = raw pixel CNN）
- 用 AutoCurriculum 确保"先简单（1 个球）→ 慢慢增加物体数量"

**与旧 Stage 0 的关系**: 复用 PPO + RND + RSSM，不重新设计。换环境、换好奇心信号源即可。

**里程碑**: 智能体主动靠近、碰撞物体。覆盖率 > 快速上升。

---

### Phase 1 · 婴儿感知运动（Sensorimotor）
**发育年龄**: 1–4 月 ｜ **参数**: 3M（+Slot Attention + 生长 1 层）｜ **训练**: ~2 周

**能力目标**:
- 物体中心感知：看到场景不是 49 个像素，是"2 个球和 1 个方块"
- 初级抓取/推动策略
- 物体属性记忆：这个球是红色的、重的

**环境**: 升级到 3D 物理沙盒（Brax / MuJoCo / SAPIEN）
- 5–10 个物体（球、方块、圆柱）
- 物体有不同的颜色、重量、弹性
- 智能体有简单的"手"（两个点接触）

**训练信号**:
- 世界模型预测不确定性（RSSM 预测误差大 → 这是有意思的状态）
- 物体分割奖励（Slot Attention 能清晰地分开物体 → 正信号）
- 接触奖励 + 推动奖励 + 抓取奖励（渐进解锁）

**发育控制**:
- Slot Attention 开始解冻（前 50k 步冻结，之后逐渐解冻）
- 增加 1 层 TTT-Hybrid 骨干（Model Growth）

**里程碑**: 智能体能把物体分开、推到指定位置。Slot Attention 的输出可以被可视化——每个槽位对应一个物体。

---

### Phase 2 · 因果发现（Causal Discovery）
**发育年龄**: 4–8 月 ｜ **参数**: 6M（+2 层骨干）｜ **训练**: ~3 周

**能力目标**:
- 从"碰 A → A 动了"升级到"推 A 撞 B → B 也动了"
- 简单工具使用（用棍子够远处的球）
- 物体遮挡推理（看不到的东西还在）

**环境**: 带因果链的物理沙盒
- 多米诺骨牌（推倒第一个 → 其他倒下）
- 斜坡（物体滚下去）
- 堆叠（A 放在 B 上面）
- 可移动障碍物

**训练信号**:
- 好奇心指向中等难度（RSSM 预测既不完美也不完全随机）
- 因果干预奖励（"你做了一个动作，导致了远处物体状态变化"）
- 新鲜度奖励（DivergentGenerator 已实现）

**发育控制**:
- 骨干部扩大 2 层（总计 4 层 TTT-Hybrid）
- 子目标策略开始解冻（Hierarchical Policy 学习率从 0 增长）

**里程碑**: 智能体推倒多米诺链。用棍子够到远处的球。在遮挡后追球。

---

### Phase 3 · 模仿与社交（Imitation）
**发育年龄**: 8–12 月 ｜ **参数**: 12M（+Hierarchical Policy 头）｜ **训练**: ~3 周

**能力目标**:
- 观察一个"看护者"做动作，模仿同样的动作
- 共同注意（看护者指一个东西 → 智能体也看那个东西）
- 简单的轮流动作

**环境**: 多智能体物理沙盒
- 看护者（脚本 / 预录轨迹）做各种动作
- 智能体观察 → 尝试复制
- 看护者可能"纠正"智能体的错误动作（reward 信号）

**训练信号**:
- Behavioral Cloning loss（已有 BehaviorCloningHead）
- 模仿奖励（action 距离看护者越近越好）
- 共同注意奖励（智能体的视线跟随看护者）

**发育控制**:
- Hierarchical Policy 完全激活
- 骨干部扩大 4 层（总计 8 层）
- 语言条件化开始微弱解冻（LanguageEncoder 从 0 开始）

**里程碑**: 智能体看到看护者推一个方块，自己去推同一个方块。看护者指一个球，智能体视线转向球。

---

### Phase 4 · 语言扎根（Language Grounding）
**发育年龄**: 12–18 月 ｜ **参数**: 25M ｜ **训练**: ~4 周

**能力目标**:
- 词语→物体映射："红色的球" = 房间里那个红球
- 简单指令执行："推方块"
- 物体+属性组合："大红色的球"（两个修饰语的组合）

**环境**: 物体 + 文本标签 / 语音指令
- 每个物体有文本描述（颜色、形状、位置）
- 看护者发出简单指令（可以是文本形式）
- 指令 + 视觉输入 → 动作

**训练信号**:
- 多模态对比学习（这个物体 = 这个词）
- 指令完成奖励（做了正确的事）
- LanguageEncoder + FiLM（已有的，但之前下载 CLIP 失败）

**注意**: 这里不用大 LLM。用一个小型 CLIP 文本编码器（~60M 冻结）或从零训练的跨模态匹配器。核心不是"语言理解"，是"符号→感知映射"。

**发育控制**:
- LanguageEncoder 解冻
- Vision FiLM 激活（语言调制视觉特征）
- 骨干部达到 12 层

**里程碑**: "拿红色的球" → 智能体走到红球旁触碰它。"把方块放在圆上面" → 堆叠正确。

---

### Phase 5 · 规划（Planning）
**发育年龄**: 18–24 月 ｜ **参数**: 50M ｜ **训练**: ~6 周

**能力目标**:
- 多步规划（5–10 步）：拿到钥匙 → 走到门前 → 开门 → 拿门后面的奖励
- 子目标分解：不在一步想全局，分层规划
- 障碍物绕行：如果直接路径被堵，找另一条路

**环境**: 复杂任务环境
- 需要多步因果链（钥匙→门→奖励）
- 可移动障碍物和动态路径
- 多个子目标同时存在（选择哪一个做）

**训练信号**:
- Hierarchical RL（高层选子目标，低层执行）
- 子目标完成奖励（每完成一个子目标给奖）
- 世界模型想象（RSSM roll out 多种可能性，挑最好的）

**关键复用**: HierarchicalActorCritic + ThoughtActionLoop + RSSM imagine_step（全部已实现）

**发育控制**:
- Model Growth 达到 16 层 TTT-Hybrid
- RSSM rollout 深度增加（限制从 1 步 → 5 步）

**里程碑**: 智能体完成"开门拿钥匙"的三步链。能绕过一个障碍物到达目标。

---

### Phase 6 · 抽象推理（Abstract Reasoning）
**发育年龄**: 2–4 岁 ｜ **参数**: 100M ｜ **训练**: ~8 周

**能力目标**:
- 类比推理：A:B :: C:D
- 简单数学：数物体 → "这里有 3 个球"
- 模式完成：红色-蓝色-红色-?

**环境**: 抽象推理任务
- 形状类比（大圆:大方的类比 → 小圆:?）
- 物体计数场景
- 序列/模式任务（视觉推理）

**训练信号**:
- 正确答案奖励
- 推理步骤正确性（不只是最终答案）
- Counterfactual 验证（如果选错了会怎样）

**关键复用**: 以下已实现但从未真正用：
- CounterfactualImagination（如果做另一个动作会怎样）
- HypothesisTester（假设检验）
- NeuralSymbolicLayer（规则抽取，之前 rules=0）

**关键在于**: 这个 Phase 需要真正让符号推理工作起来。不是余弦匹配的"if-then"，而是通过 Counterfactual + HypothesisTester 让世界模型学会因果。

**里程碑**: 完成 shape analogy。数出 3 个物体。"这个序列的下一个是?"

---

### Phase 7 · 元学习（Meta-Learning）
**发育年龄**: 4–8 岁 ｜ **参数**: 150M ｜ **训练**: ~12 周

**能力目标**:
- 快速适应新任务（从 < 100 个样本学习）
- 任务转移：学会推球 → 可以推方块（不需要重新学）
- 学会学习：在新环境中更快学习（因为以前见过类似的）

**环境**: 多样化任务分布
- 多任务并发（不只是 5 个 MiniGrid 任务）
- 同一类任务的不同参数（推不同重量的物体）
- OOD 泛化测试（见过 5 种颜色, 能推第 6 种颜色的物体吗？）

**训练信号**:
- Meta-RL（EMA 元学习器，已实现：MetaLearner）
- 迁移学习效率奖励（在新任务上收敛越快越好）

**发育控制**:
- 骨干部达到最终 20 层 TTT-Hybrid
- Slot Attention 完全成熟
- Language 和 Vision 通道深度整合

**里程碑**: 在新物理环境（从未见过）中从 < 10 个示范学会推新物体。

---

### Phase 8 · 创造性探索（Creative Exploration）⚠️ 探索性阶段

**发育年龄**: 3–5 岁等效 ｜ **参数**: 150M ｜ **训练**: 长期（数周→数月）

**诚实定位**: 此阶段以下的 Phase 0–7 有明确可实现路径。Phase 8 是开放探索——已有工具（DivergentGenerator、TransformationalCreativity、ModelGrower）可以运行，但"真正的创造力"需要符号理解能力，那是硬瓶颈。Phase 8 能做的是"随机组合 → 筛选有用的"（类似遗传算法），不是"灵光一现的洞察"。


**能力目标**:
- 在没有明确目标的开放环境中主动探索
- 组合已有技能尝试新方案（随机组合 + 筛选）
- 在没有外部奖励时维持行为（内在动机驱动）

**环境**: 开放世界沙盒
- 不定义明确的任务（智能体自己找目标）
- 有丰富的物理交互
- 周期性地看护者（教师）给出抽象指令

**训练信号**:
- 内在动机为核心（不再依赖外在奖励）
- 技能新颖性（DivergentGenerator + TransformationalCreativity）
- 教师反馈（偶尔的人工评估）

**关键复用**: 
- DivergentGenerator（随机组合 → 新颖想法）
- TransformationalCreativity（打破规则 → 根本创新）
- ModelGrower（智能体自己决定什么时候要扩大容量）

**里程碑**: 智能体在没有指令的情况下探索环境，自己发现一种新的推物体方式，然后反复使用它。

## 6. 诚实的能力天花板

| 发育年龄等效 | 能做的事 | 做不了的事 | 对标 | 可信度 |
|---|---|---|---|---|
| 新生（Phase 0）| 碰物体、追踪运动 | 无 | 0–3 月人类 | 高（1 周出结果）|
| 婴儿（Phase 1–2）| 物体分割、推拉、简单因果 | 语言、规划、记忆 | 3–12 月人类 | 高（已有 Slot Attention 论文支撑）|
| 幼儿（Phase 3–4）| 模仿、跟随指令、简单堆叠 | 抽象推理、数学、创造 | 1–3 岁人类 | 中高（BC + CLIP 是成熟技术）|
| 学龄前（Phase 5–6）| 多步规划、简单类比、物体计数 | 复杂抽象、系统思维 | 3–5 岁人类 | 中（规划可行，类比/计数部分可行）|
| 探索（Phase 7–8）| 快速适应、开放探索、技能组合 | 真正的概念理解、常识、元认知 | 聪明的乌鸦 → 5 岁人类 | 中低（硬瓶颈：神经符号桥梁）|

**为什么从"12 岁"修正到"3–5 岁"：** 之前的 Phase 7–8 描述假设了"符号推理可通过余弦匹配实现"和"世界模型可学会抽象概念"——但 §10 的差距分析表明这两者都是硬瓶颈。去掉乐观假设后，诚实天花板在 3–5 岁等效水平（具体领域可到"聪明乌鸦"级别）。

## 9. 总结

**不改架构。改环境和训练哲学。** TTT-Hybrid + RSSM + SkillLibrary + 六公理 = 正确的骨肉。缺的是"发育式训练"——让同一架构在逐渐复杂的环境中、按生物发育时序逐步成长。

## 10. 差距工程：哪些能补，哪些不能

### 10.1 可工程补上的（在 9 Phase 内逐步实现）

#### A. 训练时间 → 拉长即可

```
当前计划: 30 周 × 7×24h = 5,000 小时
拉长后:   100 周 × 7×24h = 16,800 小时（3.3×）
家用 64GB 机 7×24 跑两年 ≈ 17,500 小时（3.5×）
```

**已有基础设施**：Sleep Consolidation（夜间固化）、HealthChecker（内存泄漏监控）、六公理（长期稳定性保障）。代码层面不需要新东西，只是加 `--total-steps`。

#### B. 感官带宽 → 加输入模态

### 10.2 工程可部分弥补的（能缩小差距，不能填平）

#### H. 共享文化语境 → 多智能体交互

```
能做的: 10 个智能体同时训练，互相观察
不能做的: 共享的符号系统、隐喻、笑话——这些需要人类的
          文化演化历史，不是几个 agent 交互能产生的
```

差距系数从 ∞ 缩小到 ~100×（有了观察学习 ≠ 有了文化）。

#### I. Theory of Mind → 其他智能体预测器

```
能做的: 加一个 "他人预测器" MLP：
  输入: 观察到的他人动作 → 输出: 预测他人下一个动作
  训练信号: 与实际观察到的他人动作的 MSE
  
不能做的: 二级信念推理 ("我知道他知道我不知道的事")
          一级预测器可以学，二级需要递归符号推理
          而那跟当前的 LogicEngine 一样——余弦匹配 ≠ 逻辑
```

差距系数从 ∞ 缩小到 ~50×。

#### J. 单样本概念学习 → 更好的世界模型 + 类比

```
能做的: 
  - 通过世界模型想象 (imagine_step) 在新概念附近生成大量
    虚拟样本 → "伪单样本学习"
  - 通过类比迁移 (已见过的概念 A → 新概念 B 如果 A≈B)

不能做的:
  - 真正的单样本学习需要贝叶斯程序归纳 (Bayesian Program
    Learning)，Lake et al. 2015 Science——这是完全不同的
    算法范式，不在 TTT-Hybrid 的能力范围内
```

差距系数从 1000× 缩小到 ~100×（伪样本 + 类比 ≠ 真正的归纳）。

### 10.3 当前科学无法解决的

#### K. 真正的符号推理（合一 / 归结 / 归纳逻辑编程）

```
TTT-Hybrid 是连续向量计算。符号推理是离散符号操作。
这两者之间的桥梁（神经符号 AI）是 AI 领域 30 年未解决
的开放问题。余弦相似度匹配不是合一。

替代方案: 放弃"神经系统做符号推理"，改为:
  - 让 Slot Attention 学到的物体属性 (颜色/形状/重量)
    作为符号输入给一个小型专门的 LogicEngine
  - 但这个 LogicEngine 的规则从哪里来？
    → 目前只能人工写，或者从数据中归纳 (ILP)
    → ILP 在 >10 条规则时计算复杂度爆炸
```

**这是硬瓶颈。突破它需要 AI 基础研究，不是工程。**

#### K. 真的"理解"vs "模式匹配"

```
人类: 看到"球"不只是在视网膜上激活一圈神经元，
      还激活了"球 = 圆的、弹的、可以扔的、会滚的"这一整
      套概念网络——这是通过 15 年的多感官互动建立的。

TTT-Hybrid: 看到球激活了一个 384 维向量。这个向量跟
      其他向量在余弦空间中的关系"逼近"了概念关系，
      但不是概念本身。

这个问题叫 "符号扎根问题" (Symbol Grounding Problem)，
由 Stevan Harnad 在 1990 年提出。至今未解决。
```

#### L. 先验知识 / 天生结构

```
人类: 出生时已有很多天生结构——深度感知、面孔识别偏好、
      因果感知、数的近似感。这些不是"学"来的。

TTT-Hybrid: 完全空白初始化。连"近处的物体比远处的大"
      都不懂。

补偿: 用预训练 DINOv2 可弥补部分视觉先验。
      但其他模态 (因果、数感、物理直觉) 没有预训练模型。
```
